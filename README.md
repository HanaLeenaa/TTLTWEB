# Rule
1. Quy tắc đặt tên nhánh (Branching Strategy)
   Tuân theo cấu trúc: loại-nhánh/tên-ngắn-gọn
   feat/: Phát triển tính năng mới (vd: feat/cart-page)
   fix/ hoặc bugfix/: Sửa lỗi (vd: fix/login-validation)
   hotfix/: Sửa lỗi gấp trên nhánh maindocs/: Cập nhật tài liệu, README
   refactor/: Tối ưu hóa hoặc cấu trúc lại code nhưng không thay đổi tính năng
2. Quy chuẩn Commit Message (Conventional Commits)
   Cấu trúc commit: <type>(<scope>): <description>Loại (Type)
   feat(database): tổ chức lại bảng products và gallery
   fix(cart): sửa lỗi không tăng được số lượng sản phẩm
